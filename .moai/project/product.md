# TMK (Test My Knowledge) - 제품 문서

## 프로젝트 개요

**프로젝트 명칭**: TMK (Test My Knowledge)

**태그라인**: AI 기반 자동 문제 생성 및 지능형 시험 플랫폼

**버전**: 0.0.1-SNAPSHOT

**상태**: MVP 개발 중

## 제품 설명

TMK는 PDF 문서를 기반으로 OpenAI의 생성형 AI를 활용하여 **자동으로 시험 문제를 생성**하는 AI 기반 문제은행 플랫폼입니다. 사용자는 학습 자료를 등록한 후 자동 생성된 문제로 시험을 응시하고, 즉시 채점 및 결과 분석을 받을 수 있습니다.

### 핵심 기술적 특징

- **RAG 기반 질문 생성**: 문서를 청킹하고 OpenAI 임베딩 모델(text-embedding-3-small)로 벡터화한 후 pgvector의 ANN(Approximate Nearest Neighbor) 검색을 통해 가장 관련성 높은 내용을 추출하여 LLM에 전달
- **자동 시험 채점**: Spring Batch가 매분 만료된 시험을 감지하여 자동으로 제출 및 채점 처리
- **멀티 인증 지원**: 이메일 기반 회원가입 및 Google, Kakao, Naver 소셜 로그인
- **캐시 최적화**: Redis를 활용한 이메일 인증 코드, JWT 토큰, 시험 진행 중 임시 답안 저장

## 문제 정의

### 왜 이 서비스가 필요한가?

**학습 환경의 변화에 따른 니즈**

1. **시간 소모적인 문제 출제**: 교육 기관이나 개인 학습자가 교육 자료로부터 시험 문제를 수동으로 만드는 것은 많은 시간과 노력이 필요합니다.

2. **평가 과정의 자동화 부족**: 대면 평가에서 벗어나 온라인 환경에서도 즉각적인 피드백이 필요합니다.

3. **맞춤형 학습 평가 부재**: 학습자의 이해도를 정확히 파악할 수 있는 도구가 부족합니다.

4. **컨텐츠 기반 문제 생성 자동화**: 기존 시스템은 정형화된 문제 은행만 제공하며, 새로운 교육 자료에 맞춘 자동 문제 생성이 불가능합니다.

### 해결책

TMK는 다음과 같은 방식으로 위 문제들을 해결합니다:

- PDF 또는 텍스트 문서 업로드 → 자동 문제 생성 (시간 절감: 80% 이상)
- 다양한 문제 유형 지원 (객관식, 단답형, 참/거짓)
- 즉시 채점 및 상세 결과 분석 (학습자의 이해도 정량화)
- 난이도 기반 시험 구성 (학습 진도에 따른 맞춤형 평가)

**온라인 교육 플랫폼 (Udemy, Coursera 등 모방 서비스)**
- 요구사항: 강의 콘텐츠에 맞춘 자동 퀴즈 생성
- 기대효과: 학습자 참여도 증가, 완성도 높은 강의 제공
- 사용 시나리오: 각 수업 모듈 완료 후 자동 평가

**개인 학습자**
- 요구사항: 개인 학습 자료 기반 자가 진단 테스트
- 기대효과: 학습 이해도 정량화, 학습 동기 부여
- 사용 시나리오: 시험 준비, 자격증 취득 준비

## 핵심 기능 목록

### 1. 문서 등록 및 처리 (MVP)

- **내부 API 기반 문서 등록**: `/internal/v1/documents` 엔드포인트를 통해 시스템 관리자가 PDF 파일 등록
- **자동 처리 파이프라인**:
  - 텍스트 파싱 및 청킹 (청크 크기: 설정 가능, 기본 1024 토큰)
  - OpenAI text-embedding-3-small으로 1536차원 벡터 생성
  - pgvector (HNSW 인덱스, m=16, ef_construction=64)에 저장
  - 문서당 최소 2개 문제 자동 생성
- **상태 추적**: 등록, 처리 중, 완료, 실패 상태 관리

### 2. 사용자 인증 및 계정 관리

- **이메일 기반 회원가입**:
  - 이메일 인증 코드 발송 (5분 유효, Redis 저장)
  - 인증 후 회원가입 가능
  - 로컬 사용자: password 필수

- **소셜 로그인**:
  - Google, Kakao, Naver 지원
  - 소셜 사용자: providerId 필수

- **토큰 관리**:
  - JWT 기반 accessToken, refreshToken
  - refreshToken은 Redis에 저장 (TTL 관리)

### 3. 시험 생성 및 응시

- **시험 자동 생성**:
  - 최소 10개 문제 포함
  - 난이도별 최소 1문제 (EASY, MEDIUM, HARD)
  - 시험 시간: 30분 (기본값, 설정 가능)

- **시험 응시**:
  - 실시간 답안 저장 (임시 답안은 Redis에 저장)
  - 시험 중 진행률 표시
  - 시간 초과 알림

- **자동 제출 및 채점**:
  - Spring Batch가 매분 만료된 시험 감지
  - 자동 제출 및 즉시 채점
  - 채점 로직: 정답 매칭 기반 점수 계산

### 4. 문제 관리 및 조회

- **문제 유형**:
  - MULTIPLE_CHOICE (객관식, 보기 5개 필수)
  - SHORT_ANSWER (단답형)
  - TRUE_FALSE (참/거짓)

- **문제 속성**:
  - 난이도: EASY, MEDIUM, HARD
  - 문제 텍스트, 정답, 해설
  - 생성 근거 컨텍스트 (문서에서 추출된 원문)

- **조회 기능**:
  - 전체 문제 목록 (페이징, 필터링)
  - 문제 상세 조회

### 5. 시험 결과 분석

- **채점 결과**:
  - 총점, 정답률
  - 합격 기준: 정답률 50% 이상
  - 문제별 정답/오답 표시

- **상세 분석**:
  - 난이도별 정답률
  - 취약 분야 파악 (문제의 주제/카테고리)
  - 해설 및 학습 자료 제공

- **히스토리 관리**:
  - 시험 응시 히스토리 목록
  - 각 시험의 상세 결과 조회

### 6. 관리 기능 (향후 추가)

- 대시보드: 사용자, 문제, 시험 통계
- 배치 작업: 자동 제출 및 채점, 만료 인증 코드 삭제

## 주요 사용 시나리오

### 시나리오 1: 교사의 시험 준비

```
교사: PDF 교과서 업로드 → TMK 자동 문제 생성
     → 생성된 문제 검토 및 선택
     → 시험 생성
     → 학생에게 공개

학생: 시험 응시 → 자동 채점 → 결과 분석 → 부족한 부분 학습
```

**소요 시간**:
- 수동 출제: 2-3시간
- TMK 사용: 10-15분 (선택 및 검수만)
- **시간 절감: 85% 이상**

### 시나리오 2: 학생의 자가 진단

```
학생: 학습 자료 업로드
    → 자동 문제 생성
    → 시험 응시
    → 즉시 채점 및 해설 확인
    → 취약 부분 재학습
```

**기대효과**: 실시간 학습 진도 파악, 맞춤형 학습 가능

### 시나리오 3: 교육 기관의 대규모 평가

```
기관 관리자: 다수의 강의 자료 일괄 등록
          → 주간 자가 진단 평가 자동 생성
          → 학생별 성과 추적
          → 통계 기반 교육 개선
```

**기대효과**: 규모 있는 평가 체계 자동화, 교육 품질 개선

## 핵심 비즈니스 규칙

### 문서 관련 규칙

| 규칙 | 설명 | 이유 |
|------|------|------|
| **최소 2개 문제 생성** | 하나의 문서에서 최소 2개 이상의 문제를 생성해야 함 | 충분한 평가 문제 보장 |
| **청킹 단위 표준화** | 문서는 일정 크기의 청크로 분할되어 벡터화 (기본 1024 토큰) | 임베딩 품질 및 ANN 검색 정확도 보장 |
| **벡터 차원 고정** | 모든 청크는 1536차원 벡터로 표준화 | OpenAI text-embedding-3-small 스펙 준수 |
| **문서당 유일한 임베딩** | 동일 문서의 청크들은 동일 임베딩 모델 사용 | 일관된 벡터 공간 유지 |

### 사용자 및 인증 규칙

| 규칙 | 설명 | 이유 |
|------|------|------|
| **인증 코드 유효기간** | 이메일 인증 코드는 5분 유효 | 보안 강화, 무작위 공격 방지 |
| **로컬 사용자 패스워드 필수** | provider=LOCAL인 경우 password 필드 필수 | 계정 접근 제어 |
| **소셜 사용자 providerId 필수** | 소셜 로그인 사용자는 providerId 필수 | 소셜 계정과 로컬 계정 분리 |
| **중복 이메일 불가** | 동일 이메일로 중복 가입 불가 (소셜 포함) | 계정 통합 및 보안 |
| **RefreshToken Redis 저장** | RefreshToken은 DB 저장 금지, Redis만 사용 | 빠른 토큰 검증, 무효화 용이 |

### 문제 관련 규칙

| 규칙 | 설명 | 이유 |
|------|------|------|
| **객관식 보기 고정** | MULTIPLE_CHOICE는 정확히 5개 보기 필수 | 일관된 평가 기준 |
| **단답형/참거짓 보기 없음** | SHORT_ANSWER, TRUE_FALSE는 options 필드 없음 | 문제 유형별 구조 명확화 |
| **문제별 난이도** | 모든 문제는 EASY/MEDIUM/HARD 중 하나 | 시험 난이도 구성 용이 |
| **생성 맥락 저장** | 문제 생성 시 사용된 청크(컨텍스트) 저장 | 문제의 신뢰성 추적, 학습자의 이해 도움 |

| 규칙 | 설명 | 이유 |

| **난이도별 최소 1문제** | 각 난이도(EASY, MEDIUM, HARD)에서 최소 1문제 | 균형 잡힌 평가 |
