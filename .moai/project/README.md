# TMK 프로젝트 문서 인덱스

TMK (Test My Knowledge) 프로젝트의 종합 문서 모음입니다. 
모든 문서는 한국어로 작성되었습니다.

## 문서 목록

### 1. [product.md](./product.md) - 제품 사양 문서

**대상 독자**: 프로젝트 관리자, 기획자, 비즈니스 담당자

**주요 내용**:
- 프로젝트 개요 및 태그라인
- 문제 정의 (왜 이 서비스가 필요한가?)
- 대상 사용자 (3가지 주 사용자군)
- 핵심 기능 목록 (6가지 기능 영역)
- 주요 사용 시나리오 (3가지 상세 사용 사례)
- 핵심 비즈니스 규칙 (80개 이상의 규칙)
- 구현 상태 현황 (완료/부분/미구현)
- 성공 지표 및 KPI

**파일 크기**: 13KB (342줄)

### 2. [structure.md](./structure.md) - 프로젝트 구조 문서

**대상 독자**: 개발자, 아키텍트, 기술 리드

**주요 내용**:
- 전체 프로젝트 디렉토리 구조
- 3개 모듈의 상세 패키지 구조
  - tmk-core (도메인 모듈): 도메인 엔티티 및 포트 정의
  - tmk-api (REST API 모듈): 헥사고날 아키텍처 구현
  - tmk-batch (Spring Batch 모듈): 배치 작업 정의
- 패키지 명명 규칙 (11가지 클래스 타입)
- 의존성 아키텍처 (빌드 시점 및 런타임)
- 모듈별 책임 매트릭스 (11개 기능 영역)
- 주요 파일 위치 참조표
- 모듈 간 통신 흐름 (REST API 및 배치)
- 확장 가능성 및 향후 개선 사항

**파일 크기**: 23KB (623줄)

### 3. [tech.md](./tech.md) - 기술 스택 문서

**대상 독자**: 개발자, 데이터베이스 엔지니어, DevOps 엔지니어

**주요 내용**:
- 기술 스택 개요 (9개 카테고리, 30개 항목)
- 프레임워크 선택 이유 및 대안 검토
  - Spring Boot 3.5.11 선택 이유
  - PostgreSQL + pgvector 선택 이유
  - JWT 인증 선택 이유
- PostgreSQL + pgvector 데이터베이스 설계
  - 6개 테이블 DDL
  - HNSW 벡터 검색 인덱스 (m=16, ef_construction=64)
  - 인덱스 전략 및 성능 특성
- Redis 설계 및 키 전략
- OpenAI API 통합 설계
  - text-embedding-3-small (1536차원)
  - GPT-4 Turbo vs GPT-3.5 비용/성능 분석
  - RAG 플로우 5단계
- 보안 아키텍처 (JWT, Spring Security)
- 성능 최적화 (벡터 검색, 쿼리, 캐싱)
- 개발 환경 요구사항 및 설정
- 빌드 및 실행 명령어
- 프로덕션 배포 구성
- 기술 결정 문서 (ADR)

**파일 크기**: 23KB (837줄)

## 빠른 참조

### 개발자를 위한 시작 가이드

1. **프로젝트 이해하기**
   - [product.md](./product.md) 읽기 → 프로젝트 개요 파악
   - [structure.md](./structure.md) 읽기 → 코드 구조 이해

2. **개발 환경 설정**
   - [tech.md](./tech.md)의 "개발 환경 요구사항" 섹션 참고
   - Java 21 설치
   - PostgreSQL + pgvector 설치
   - Redis 설치

3. **첫 빌드 및 실행**
   - [tech.md](./tech.md)의 "빌드 및 실행 명령어" 섹션 참고
   - `./gradlew build`
   - `./gradlew :tmk-api:bootRun`

### 아키텍트를 위한 설계 검토

1. **의존성 아키텍처** → [structure.md](./structure.md)의 "의존성 아키텍처" 섹션
2. **기술 선택 근거** → [tech.md](./tech.md)의 "프레임워크 선택 사유" 섹션
3. **모듈 책임** → [structure.md](./structure.md)의 "모듈별 책임 매트릭스" 섹션
4. **성능 최적화** → [tech.md](./tech.md)의 "성능 최적화" 섹션

### 프로젝트 관리자를 위한 개요

1. **프로젝트 상태** → [product.md](./product.md)의 "구현 상태" 섹션
2. **핵심 기능** → [product.md](./product.md)의 "핵심 기능 목록" 섹션
3. **비즈니스 규칙** → [product.md](./product.md)의 "핵심 비즈니스 규칙" 섹션
