# TMK (Test My Knowledge) 도메인 모델 설계

> 작성일: 2026-04-27
> 버전: v2.0.0
> 아키텍처: 클린 아키텍처 기반 멀티 모듈

---

## 멀티 모듈 구조

```text
tmk-parent/
├── tmk-core/               # 도메인 엔티티, 애플리케이션 서비스, outbound port
├── tmk-infra/              # JPA repository, persistence adapter
├── tmk-admin/              # 관리자 전용 controller/usecase/dto
├── tmk-api/                # 단일 Boot Application, 사용자 API, admin 모듈 포함 기동
└── tmk-batch/              # 시험 자동 제출 배치
```

`tmk-admin`은 독립 실행 애플리케이션이 아니라 라이브러리 모듈이며, `tmk-api` 서버가 기동될 때 함께 로드되어 관리자 API를 같은 서버에서 제공합니다.

## 도메인 목록

| 도메인 | Aggregate Root | 설명 |
|--------|---------------|------|
| User | User | 사용자 계정, 권한, 국가 코드 |
| Topic | Topic | 공용 문제 분류 주제 |
| Document | Document | 사용자 문서 메타데이터와 처리 상태 |

| Exam | Exam | 시험 세션, 답안, 채점 |

## 1. User 도메인

### Aggregate Root: User

```text
User
├── id: Long
├── username: String
├── password: String
├── role: UserRole
├── countryCode: String
├── createdAt: OffsetDateTime
└── updatedAt: OffsetDateTime
```

### UserRole

```text
USER
ADMIN
```

### 도메인 규칙

- `username`은 시스템 내에서 유일해야 한다.
- 회원가입은 아이디/비밀번호 기반으로만 처리한다.
- `countryCode`는 문제/정답/해설 생성 언어를 결정하는 입력값이다.
- `ADMIN` 계정은 기존 `ADMIN`만 생성할 수 있다.

### Outbound Port

```java
public interface UserPort {
    User save(User user);
    Optional<User> findById(Long id);
    Optional<User> findByUsername(String username);
    boolean existsByUsername(String username);
}
```

## 2. Topic 도메인

### Aggregate Root: Topic

```text
Topic
├── id: Long
├── name: String
├── description: String
├── active: boolean
├── createdBy: Long
├── createdAt: OffsetDateTime
└── updatedAt: OffsetDateTime
```

### 도메인 규칙

- Topic은 admin만 생성/수정할 수 있다.
- Topic 이름은 유일해야 한다.
- 비활성 Topic은 새 공용 문제 등록이나 공용 시험 시작 대상에서 제외된다.

### Outbound Port

```java
public interface TopicPort {
    Topic save(Topic topic);
    Optional<Topic> findById(Long id);
    List<Topic> findAllActive();
    boolean existsByName(String name);
}
```

## 3. Document 도메인

### Aggregate Root: Document

```text
Document
├── id: Long
├── userId: Long
├── title: String
├── sourceType: DocumentSourceType
├── sourceReference: String
├── status: DocumentStatus
├── generatedQuestionCount: int
├── createdAt: OffsetDateTime
└── updatedAt: OffsetDateTime
```

### Entity: DocumentChunk

```text
DocumentChunk
├── id: Long
├── documentId: Long
├── chunkIndex: int
├── content: String
└── embedding: float[]
```

### Enums

```text
DocumentSourceType
├── PDF_UPLOAD
├── NOTION
└── URL

DocumentStatus
├── PROCESSING
├── COMPLETED
└── FAILED
```

### 도메인 규칙

- 사용자는 자신의 문서를 직접 등록할 수 있다.
- 파일 업로드는 PDF만 허용한다.
- 노션/URL은 서버가 실제로 읽을 수 있을 때만 등록할 수 있다.
- 원본 문서는 문제 생성 완료 후 서버 저장소에서 삭제하고, 도메인에는 메타데이터만 남긴다.
- 문서당 최소 2개 이상의 문제가 생성되어야 한다.

### 문서 처리 플로우

```text
문서 등록
→ 텍스트 추출
→ 청킹
→ 임베딩 생성
→ 유사 청크 기반 문제 생성
→ 문제 저장
→ 원본 문서 삭제
→ 상태 COMPLETED
```

### Outbound Port

```java
public interface DocumentPort {
    Document save(Document document);
    Optional<Document> findById(Long id);
    List<Document> findByUserId(Long userId);
}

public interface DocumentChunkPort {
    List<DocumentChunk> saveAll(List<DocumentChunk> chunks);
    void deleteByDocumentId(Long documentId);
}
```

## 4. Question 도메인

### Aggregate Root: Question

```text
Question
├── id: Long
├── ownerUserId: Long?          // 개인 문제일 때만 값 존재
├── topicId: Long?              // 공용 문제일 때만 값 존재
├── documentId: Long?           // 문서 기반 문제일 때만 값 존재
├── createdBy: Long?            // 공용 문제를 등록한 admin
├── scope: QuestionScope
├── sourceType: QuestionSourceType
├── content: String
├── type: QuestionType
├── difficulty: Difficulty
├── answer: String
├── explanation: String
├── languageCode: String
├── options: List<QuestionOption>
├── createdAt: OffsetDateTime
└── updatedAt: OffsetDateTime
```

### Entity: QuestionOption

```text
QuestionOption
├── id: Long
├── questionId: Long
├── optionNumber: int
└── content: String
```

### Enums

```text
QuestionScope
├── PUBLIC
└── PRIVATE

QuestionSourceType
├── PUBLIC_MANUAL
└── PRIVATE_DOCUMENT_GENERATED

QuestionType
├── MULTIPLE_CHOICE
├── SHORT_ANSWER
└── TRUE_FALSE

Difficulty
├── EASY
├── NORMAL
└── HARD
```

### 도메인 규칙

- 공용 문제는 admin만 직접 등록할 수 있다.
- 공용 문제 등록 시 `content`, `answer`, `explanation`, `difficulty`, `topicId`가 필수다.
- 개인 문제는 사용자 문서에서 AI가 생성한다.
- 문제/정답/해설 생성 언어는 사용자 `countryCode`에서 파생된다.
- `MULTIPLE_CHOICE`는 선택지가 반드시 5개여야 한다.
- `TRUE_FALSE`는 선택지가 반드시 2개여야 하며 자유 입력 답안을 받지 않는다.
- `SHORT_ANSWER`는 서술형이 아니어야 하며, 문서에 실제 존재하는 명시적 문자열/내용이 정답이어야 한다.
- `SHORT_ANSWER`의 정답 판정은 정확 일치 기준을 전제로 한다.

### Outbound Port

```java
public interface QuestionPort {
    Question save(Question question);
    List<Question> saveAll(List<Question> questions);
    Optional<Question> findById(Long id);
    List<Question> findAllByIds(List<Long> ids);
    List<Question> findByTopicId(Long topicId);
    List<Question> findByDocumentId(Long documentId);
    Map<Difficulty, List<Question>> findGroupedPublicByTopic(Long topicId);
    Map<Difficulty, List<Question>> findGroupedPrivateByDocument(Long documentId, Long userId);
}
```

## 5. Exam 도메인

### Aggregate Root: Exam

```text
Exam
├── id: Long
├── userId: Long
├── topicId: Long?
├── documentId: Long?
├── sourceType: ExamSourceType
├── totalQuestions: int
├── timeLimit: int
├── status: ExamStatus
├── startedAt: OffsetDateTime
├── expiredAt: OffsetDateTime
├── submittedAt: OffsetDateTime?
├── examQuestions: List<ExamQuestion>
└── createdAt: OffsetDateTime
```

### Entity: ExamQuestion

```text
ExamQuestion
├── id: Long
├── examId: Long
├── questionId: Long
├── orderNum: int
├── myAnswer: String?
└── isCorrect: Boolean?
```

### Enums

```text
ExamSourceType
├── PUBLIC_TOPIC
└── PRIVATE_DOCUMENT

ExamStatus
├── CREATED
├── IN_PROGRESS
└── SUBMITTED
```

### 도메인 규칙

- 사용자는 `공용 Topic 문제` 또는 `개인 문서 문제` 중 하나를 선택해 시험을 시작한다.
- 시험 생성 시 `문제 수`와 `시험 시간(분)`을 직접 입력한다.
- 시험 생성 시점에는 `CREATED` 상태로 저장되고, 별도의 시작 동작이 있어야 `IN_PROGRESS`로 전이된다.
- 시험 시간은 분 단위 양수여야 한다.
- 시험 시간이 만료되면 답안 저장/수정/수동 제출이 불가하다.
- 배치는 만료된 `IN_PROGRESS` 시험을 자동 제출한다.
- 합격 기준은 정답률 50% 이상이다.

### 도메인 서비스

#### ExamCreationService

```text
ExamCreationService
├── createPublicTopicExam(userId, topicId, questionCount, timeLimit)
└── createPrivateDocumentExam(userId, documentId, questionCount, timeLimit)
```

동작:
- 출처 타입에 따라 문제 풀을 분리해서 조회한다.
- 요청한 문제 수를 기준으로 난이도 균형을 맞춰 문제를 구성한다.
- 생성 단계에서는 `startedAt`, `expiredAt`를 비워둔 `CREATED` 시험을 만든다.

#### ExamStartService

```text
ExamStartService
└── start(exam)
```

동작:
- `CREATED` 상태의 시험만 시작 가능하다.
- 시작 시점에 `startedAt`, `expiredAt`를 계산한다.
- 상태를 `IN_PROGRESS`로 전이한다.

#### ExamGradingService

```text
ExamGradingService
└── grade(exam, questions)
```

동작:
- 객관식/참거짓형은 선택지 값으로 채점한다.
- 단답형은 정답 문자열과 정확 일치 기준으로 채점한다.

### Outbound Port

```java
public interface ExamPort {
    Exam save(Exam exam);
    Optional<Exam> findByIdAndUserId(Long id, Long userId);
    List<Exam> findByUserIdOrderByCreatedAtDesc(Long userId);
    List<Exam> findExpiredInProgressExams(OffsetDateTime now);
}
```

## 주요 플로우

### 1. 사용자 문서 기반 문제 생성 플로우

```text
회원가입(국가 코드 포함)
→ 로그인
→ PDF 업로드 또는 읽기 가능한 노션/URL 등록
→ 문서 처리
→ 사용자 국가 코드 기반 언어로 문제/정답/해설 생성
→ 원본 문서 삭제
→ 개인 문제 저장
```

### 2. 공용 문제 운영 플로우

```text
ADMIN 로그인
→ Topic 생성/관리
→ 공용 문제 직접 등록
→ 일반 사용자가 Topic 선택
→ 공용 문제 기반 시험 생성
```

### 3. 시험 응시 플로우

```text
시험 출처 선택(공용 Topic / 개인 문서)
→ 문제 수 입력
→ 시험 시간 입력(분)
→ 시험 생성
→ 시작하기
→ 답안 저장/수정
→ 제출 또는 시간 만료
→ 채점/결과 조회
```
