# 사용자 웹 요구사항

## 문서 목적

`tmk-user-web`에서 구현할 사용자용 화면 요구사항을 정리한다.

## 공통 원칙

- 대상 사용자: 일반 사용자
- 인증 방식: 아이디/비밀번호 + JWT
- 언어 생성 기준: 회원가입 시 저장한 국가 코드
- 문서 원본은 문제 생성 후 서버 저장소에서 삭제

## 화면 목록

### 1. 인증

정리할 항목:

- 회원가입 화면
- 로그인 화면
- 로그아웃 흐름
- 토큰 재발급 처리 방식

기록할 내용:

- 경로
- 입력값
- 유효성 검증
- 성공/실패 UX
- 연동 API

### 2. 홈

정리할 항목:

- 서비스 소개
- 문제 생성 진입
- 시험 진입
- 최근 학습 상태 요약

### 3. 문서 등록

정리할 항목:

- PDF 업로드
- 노션/특정 URL 등록
- 처리 가능 여부 안내
- 문제 생성 요청

### 4. 개인 문제 생성

정리할 항목:

- 생성 상태 표시
- 생성 결과 목록
- 문제 유형별 표현 방식

### 5. 시험

정리할 항목:

- 공용 Topic 기반 시험 생성
- 개인 문서 기반 시험 생성
- 문제 수 입력
- 시험 시간(분) 입력
- 시험 시작
- 문제 풀이
- 제출
- 결과 확인

### 6. 마이페이지

정리할 항목:

- 내 정보
- 국가 코드 표시
- 시험 이력
- 생성한 문제 목록

## 라우트 정리 템플릿

각 화면은 아래 형식으로 정리한다.

```md
### 화면명
- 경로:
- 목적:
- 진입 조건:
- 사용 API:
- 주요 입력값:
- 검증 규칙:
- 성공 상태:
- 예외 상태:
- 완료 기준:
```

## 비고

- 사용자 웹 요구사항은 기능 요구사항 중심으로 적는다.
- 컴포넌트 분리, 상태관리, 폴더 구조 같은 구현 세부는 별도 설계 문서에서 다룬다.
