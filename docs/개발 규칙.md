# TMK 개발 규칙

---

## API 버전 관리

- API 버전은 URL 경로에 명시한다.
- 자원 경로는 컨트롤러 상단 `@RequestMapping`에 둔다.
- 버전은 메서드 레벨 매핑에서 `ApiVersion` 상수로 관리한다.

예시:

```java
@RequestMapping("/admin/auth")
public class AdminAuthController {

    @PostMapping(ApiVersion.V1 + "/login")
    public ResponseEntity<ApiResponse<AdminLoginResponse>> login(...) {
        ...
    }
}
```

원칙:

- `"/v1"` 같은 문자열을 컨트롤러마다 직접 반복해서 쓰지 않는다.
- 하위 호환이 깨지는 변경이 생기기 전까지는 기존 버전을 유지한다.
- 새로운 버전이 필요하면 `ApiVersion.V2`를 추가하고, 구버전과 일정 기간 공존시킨다.

---

## 나중에 할 운영 작업

웹 클라이언트와 API 버전 전환을 안전하게 운영하는 작업은 가장 나중에 진행한다.

대상:

- 웹 클라이언트 버전 체크
- 구버전 프론트 강제 새로고침 유도
- `v1`, `v2` 공존 기간 운영 정책
- 구버전 API 제거 시점과 공지 정책

이 작업은 현재 구현 우선순위가 아니라, 핵심 API 개발이 끝난 뒤 운영 안정화 단계에서 진행한다.
