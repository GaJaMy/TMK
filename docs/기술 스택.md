## 프레임워크 선택 이유

### Spring Boot 3.5.11

비교 옵션: Quarkus, Micronaut, Play Framework

| 항목 | Spring Boot | Quarkus | Micronaut |
|------|-------------|---------|-----------|
| 생산성 | ✅ 최고 | 보통 | 보통 |
| 생태계 | ✅ 최대 | 중간 | 작음 |
| 메모리 | 보통 | ✅ 낮음 | ✅ 낮음 |
| 한국 커뮤니티 | ✅ 최대 | 소수 | 소수 |
| LTS 지원 | ✅ 2026년까지 | - | - |

**선택 결정**: 개발 속도와 생태계 우선 (메모리 오버헤드는 수용 가능)

### PostgreSQL + pgvector

비교 옵션: MySQL, Pinecone, Elasticsearch

| 항목 | PostgreSQL+pgvector | Pinecone | Elasticsearch |
|------|---------------------|----------|---------------|
| 월간 비용 | $0~100 (자체 호스팅) | $100+ | $500+ |
| 트랜잭션 지원 | ✅ ACID | ❌ | 제한적 |
| 벡터 검색 성능 | 우수 (HNSW) | 최고 | 우수 |
| 관계형 데이터 통합 | ✅ | ❌ | 제한적 |
| 자체 호스팅 | ✅ | ❌ | ✅ |

**선택 결정**: 관계형(사용자, 시험) + 벡터(임베딩) 통합 관리, MVP 비용 최소화

### JWT 인증

비교 옵션: OAuth 2.0 Session, API Key

| 항목 | JWT | Session |
|------|-----|---------|
| 무상태 | ✅ | ❌ |
| 수평 확장 | ✅ | 복잡 |
| 모바일 친화 | ✅ | 보통 |

**토큰 전략**:
- **AccessToken**: 30분 유효, 요청 헤더 포함
- **RefreshToken**: 7일 유효, Redis 저장

## pgvector 설정

```sql
-- pgvector 확장 활성화 (최초 1회)
CREATE EXTENSION IF NOT EXISTS vector;

-- HNSW 인덱스 (코사인 유사도 ANN 검색)
CREATE INDEX idx_document_chunk_embedding_hnsw ON document_chunk
    USING hnsw (embedding vector_cosine_ops)
    WITH (m = 16, ef_construction = 64);
```

| 파라미터 | 값 | 의미 |
|----------|-----|------|
| `m` | 16 | 각 노드의 연결 수 (높을수록 정확, 메모리 증가) |
| `ef_construction` | 64 | 인덱스 빌드 시 탐색 범위 (높을수록 정확, 빌드 느림) |

## 개발 환경 설정

### 필수 요구사항

| 도구 | 버전 | 설치 확인 |
|------|------|-----------|
| JDK | 21+ | `java -version` |
| Gradle | 8.x (wrapper 포함) | `./gradlew --version` |
| PostgreSQL | 14+ | `psql --version` |
| Redis | 7.x | `redis-cli ping` |

### PostgreSQL 초기 설정

```sql
-- 데이터베이스 및 사용자 생성
CREATE USER tmk WITH PASSWORD 'tmk';
CREATE DATABASE tmk OWNER tmk;

-- pgvector 확장 설치 (tmk DB에서 실행)
\c tmk
CREATE EXTENSION IF NOT EXISTS vector;
```

### 환경 변수

`.env` 또는 OS 환경 변수로 설정:

```bash
# 데이터베이스
DB_USERNAME=tmk
DB_PASSWORD=tmk

# Redis
REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_PASSWORD=             # 비밀번호 없으면 빈값

# JWT (256비트 이상 랜덤 문자열)
JWT_SECRET=tmk-secret-key-must-be-at-least-256-bits-long-for-hs256-algorithm

# OpenAI
OPENAI_API_KEY=sk-...
```

### 빌드 및 실행

```bash
# 전체 빌드
./gradlew build

# API 서버 실행 (기본 포트 8080)
./gradlew :tmk-api:bootRun

# 로컬 프로파일로 실행 (show-sql=true, ddl-auto=update)
./gradlew :tmk-api:bootRun --args='--spring.profiles.active=local'

# 전체 테스트
./gradlew test

# 특정 테스트 클래스 실행
./gradlew test --tests "com.tmk.api.SomeTest"

# 빌드 캐시 무시하고 테스트
./gradlew test --rerun-tasks

# Batch 서버 실행
./gradlew :tmk-batch:bootRun
```

### Spring 프로파일

| 프로파일 | `spring.jpa.hibernate.ddl-auto` | `show-sql` | 용도 |
|----------|----------------------------------|------------|------|
| default | `validate` | false | 운영 환경 |
| local | `update` | true | 로컬 개발 |

## 모듈별 의존성 구조

```
tmk-api (bootJar)
  ├── tmk-core
  ├── tmk-infra
  ├── spring-boot-starter-web
  ├── spring-boot-starter-security
  ├── spring-boot-starter-data-jpa
  ├── spring-boot-starter-data-redis
  ├── spring-boot-starter-validation
  └── jjwt (0.12.6)

tmk-infra (jar)
  ├── tmk-core
  ├── spring-data-jpa
  └── postgresql

tmk-core (jar)
  ├── spring-boot-starter-data-jpa
  ├── spring-boot-starter-data-redis
  ├── jjwt-api
  └── postgresql (JDBC driver)

tmk-batch (bootJar)
  ├── tmk-core
  ├── tmk-infra
  └── spring-boot-starter-batch
```

> 의존성 방향: `tmk-api` → `tmk-core`, `tmk-infra` / `tmk-batch` → `tmk-core`, `tmk-infra`
> `tmk-core`는 외부 프레임워크에 의존하지 않는 순수 도메인 모듈
