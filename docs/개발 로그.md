# TMK 개발 로그

> 목적: 커밋 단위 또는 작업 단위로 무엇을 수정했고 무엇을 개발했는지 기록한다.
> 원칙: 해당 시점에 실제로 반영된 작업만 기록한다.

---

## 2026-04-28

### 요구사항 및 설계 문서 재정리

- 사용자 인증, 관리자 기능, 문서 업로드, 시험 흐름 요구사항을 현재 서비스 방향에 맞게 다시 정리했다.
- ERD, DDL, 도메인 모델, API 명세를 새 구조 기준으로 다시 맞췄다.
- API 버전 관리를 `ApiVersion` 상수 방식으로 가져가는 규칙과 관련 문서를 추가했다.

### 관리자 웹 / 사용자 웹 프로토타입 정리

- 관리자 웹 로그인, 모니터링, 공용문제 관리, 관리자 관리 화면을 현재 요구사항에 맞게 정리했다.
- 사용자 웹 랜딩, 로그인, 문서 등록, 시험 보기, 시험 진행, 시험 내역 화면을 정적 프로토타입으로 구성했다.
- 기존 정적 리소스 흐름 대신 새 웹 화면 기준으로 문서와 화면 구성을 정리했다.

### 코어 / 인프라 재구성

- `tmk-core`를 현재 문서 기준으로 재구현했다.
- 일반 사용자/관리자/Topic/문서/개인 문제/공용 문제/시험/모니터링 엔티티를 새 요구사항에 맞게 정리했다.
- 기존 이메일 인증, 소셜 로그인, 단일 `User`, 단일 `Question` 구조는 제거했다.
- `tmk-infra`의 JPA repository, persistence adapter를 새 포트 구조에 맞게 다시 작성했다.

### 관리자 로그인 API 추가

- 관리자 로그인용 코어 서비스 `AdminLoginService`를 추가했다.
- `POST /admin/auth/v1/login` 엔드포인트를 추가했다.
- 관리자 로그인 시 JWT access/refresh token 발급과 Redis refresh token 저장 흐름을 연결했다.
- JWT claim 구조를 `principalId`, `principalType`, `role` 기준으로 정리했다.

### 개발 기록 방식 정리

- 커밋 시점의 실제 작업만 기록하는 `개발 로그` 문서를 추가했다.
- `commit` 스킬이 커밋 전에 `개발 로그`를 같이 갱신하도록 규칙을 추가했다.

### 관리자 로그인 인증 흐름 정리

- 관리자 로그인을 DB 직접 조회 방식이 아니라 Spring Security `AuthenticationManager` 기반 인증 흐름으로 변경했다.
- `AdminUserDetailsService`를 추가해 관리자 계정을 `UserDetailsService`로 로드하도록 바꿨다.
- 관리자 로그인 컨트롤러에 `ControllerDocs` 인터페이스 패턴을 적용하고 Swagger 어노테이션을 인터페이스로 분리했다.
- 작업 중 비어 있는 패키지 디렉터리와 중복 getter 같은 불필요 코드를 함께 정리했다.

## 2026-04-29

### 관리자 계정 관리 API 추가

- 관리자 계정 목록 조회, 생성, 상태 변경, 삭제 API를 `admin/account` 패키지 구조로 추가했다.
- 관리자 계정 생성 시 현재 로그인한 관리자의 `principalId`를 `createdByAdminId`로 저장하도록 연결했다.
- 관리자 계정 상태 변경과 삭제 시 존재 여부를 검증하고 `ADMIN_NOT_FOUND` 예외를 반환하도록 정리했다.

### 관리자 인증 principal 구조 정리

- `CustomUserDetails`를 `AuthenticatedPrincipal`로 변경해 관리자와 일반 사용자에 공통으로 쓸 수 있는 인증 주체 모델로 정리했다.
- `AdminUserDetailsService`를 `AdminAccountDetailsService`로 변경해 관리자 계정 로더 역할이 드러나도록 맞췄다.
- `/admin/**` 요청에서는 JWT의 `principalType`이 `ADMIN`인지 추가로 검증하도록 필터를 보강했다.
