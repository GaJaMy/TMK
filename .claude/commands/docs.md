코드 변경사항을 분석하여 `docs/` 폴더의 관련 문서들을 최신 상태로 업데이트합니다.

## 1. 변경사항 파악

다음을 병렬로 실행하세요:
- `git diff HEAD --stat` — 변경된 파일 목록
- `git diff HEAD` — 변경 내용 전체

## 2. docs 폴더 문서 목록 확인

`docs/` 폴더의 모든 파일을 읽어 현재 상태를 파악하세요.

## 3. 업데이트 판단 기준

각 문서별로 업데이트가 필요한 변경사항은 다음과 같습니다:

### `docs/API 명세서.md`
업데이트 트리거:
- Controller의 URL 경로 변경
- Request/Response DTO 필드 추가/수정/삭제
- 에러 응답 코드 변경 (`ErrorCode.java` 변경 포함)
- `SecurityConfig`의 permitAll 경로 변경
- 새로운 API 엔드포인트 추가

### `docs/에러코드.md`
업데이트 트리거:
- `ErrorCode.java` 에 enum 상수 추가/수정/삭제
- 에러 메시지 또는 HTTP 상태 코드 변경

### `docs/도메인 모델 설계.md`
업데이트 트리거:
- 모듈 구조 변경 (새 패키지/클래스 추가)
- 도메인 엔티티 필드 추가/수정
- UseCase 인터페이스 추가/변경
- Repository 인터페이스 변경
- 도메인 규칙 변경

### `docs/ERD 설계.md` / `docs/ddl.sql`
업데이트 트리거:
- JPA Entity 필드 추가/수정/삭제
- 테이블 구조 변경
- 새로운 인덱스 추가

### `docs/TMK(Test My Knowledge).md`
업데이트 트리거:
- 기술 스택 변경 (새 라이브러리 추가 등)
- 아키텍처 구조 변경

## 4. 업데이트 원칙

- **코드가 진실(source of truth)** 입니다. 코드를 기준으로 문서를 맞추세요.
- 문서에 없는 내용을 추가할 때는 기존 문서 스타일을 유지하세요.
- 삭제된 내용은 문서에서도 제거하세요.
- 변경이 필요 없는 문서는 건드리지 마세요.

## 5. 완료 후

업데이트한 문서 목록과 각 문서에서 변경된 내용을 요약해서 보고하세요.
업데이트가 필요한 문서가 없으면 "변경 필요한 문서 없음"으로 보고하세요.
